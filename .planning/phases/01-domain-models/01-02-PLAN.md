<!-- Mermaid classDiagram syntax rules (from 01-RESEARCH.md) -->

Required syntax rules:
- Fenced block: ```mermaid ... ```
- Attribute format: Type fieldName (e.g., "str name", "int available_hours")
- Method format: returnType methodName() (e.g., "Schedule generate_schedule()", "str explain()")
- Relationship arrows: --> (association), *-- (composition)
- Multiplicity labels MUST be quoted: "1", "many"
- Generic type notation uses ~ not <>: list~Pet~, list~Task~, list~TimeBlock~
- Do NOT use <> for generics — Mermaid rejects it

Six classes to include (from CONTEXT.md locked decisions):
1. Owner — name (str), available_hours (int), pets (list~Pet~)

3. Task — title (str), duration_minutes (int), priority (str), frequency (int), pet (Pet)
4. Scheduler — owner (Owner), tasks (list~Task~), generate_schedule() Schedule
5. Schedule — blocks (list~TimeBlock~), explain() str
6. TimeBlock — task (Task), start_time (time), end_time (time), reason (str)

Note: Scheduler, Schedule, TimeBlock are design artifacts (not yet implemented). They appear in the diagram per DEL-02 and 01-CONTEXT.md locked decision.

Also include module-level function in diagram comment or as a note:

- Schedule "1" *-- "many" TimeBlock : contains
